## web端功能列表：

### 基础界面：

1. 打开图像/显示图像界面（实际为上传），需要有上传提示进行占位（如果不是PNG，则也要进度包含上传与下载）
2. 图像转换模块（若干个按钮，点击后转换为进度，服务端先传来最近10次转换的平均时间以供进度显示，进度条最多显示90%，当服务端处理完成后在400ms内过渡到100%，且在过度时启动浏览器下载）
3. 图像增强模块（客户端进行裁剪、缩放，并将对于图像的操作的参数传给服务端，服务端操作，点击保存后显示进度条，同上为最近10次时间）
4. ai分析按钮，整体样式与打开图像类似，未分析时仅显示按钮（点击分析无需向服务端上传，使用服务端已有的内容）
5. 图像对比功能（打开后，分析界面缩小为按钮，此界面从按钮放大为窗口）（纯客户端实现，与原始图像对比，缩放锁定、x轴、y轴平移锁定）
6. 三维功能（可以显示多个模型，有快捷按钮，三维模型格式为glb，使用threejs处理）

### ai界面：

1. 聊天界面
2. ToDo List

## web端开发流程

1. 除Ai以外功能全部开发在一页中（基础界面），后续进行删减与分叉以便代码复用，第一版不分页（不包括ai界面），后续可选整合与分页模式
2. Ai功能单独开发在Ai界面中（Ai界面），同上
3. 主界面后续再开发（仅作route作用），框架界面后续再开发（界面导航，可以添加Log功能）

## 多用户同时操作怎么办？

以文件为基准，不在意用户是谁在操作，所有请求都带有xxh3以追踪文件
（后期有可能加入账号系统，但只需要token即可，添加登出功能以供用户登出，token过期问题即可无需考虑，权限细化仅需json中分配，加上管理员界面管理这些json即可）
